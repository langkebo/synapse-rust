# 数据库审计检查清单

## 表结构一致性
- [x] 所有数据库表都有对应的 Rust 结构体定义
- [x] 结构体字段名与表列名一致（或通过 serde 别名映射）
- [x] 数据类型兼容（BIGINT 对应 i64，BOOLEAN 对应 bool 等）
- [ ] ⚠️ 部分表在代码中引用但未在迁移脚本中定义（14个表）

## 时间戳字段规范
- [x] 数据库中无 created_at 字段（应使用 created_ts）
- [x] 数据库中无 updated_at 字段（应使用 updated_ts）
- [x] 代码中所有时间戳字段使用 *_ts 后缀
- [ ] ⚠️ 部分 SQL 查询和 JSON 输出仍使用旧字段名（expires_at）

## 布尔字段规范
- [x] 数据库中核心布尔字段使用 is_* 前缀
- [ ] ⚠️ 代码中约 100+ 处布尔字段未使用 is_* 前缀
- [x] API 响应使用 serde alias 保持兼容性

## 字段约束一致性
- [x] NOT NULL 约束与代码中非 Option 类型对应
- [x] NULL 允许与代码中 Option 类型对应
- [x] UNIQUE 约束在代码中有相应验证
- [x] FOREIGN KEY 约束正确关联到父表

## SQL 查询验证
- [x] 所有 SELECT 语句字段名与表结构匹配
- [ ] ⚠️ 部分 INSERT 语句使用旧字段名（expires_at, revoked_at）
- [ ] ⚠️ 部分 UPDATE 语句使用旧字段名
- [ ] ⚠️ 发现使用已废弃字段名的查询

## 代码兼容性
- [x] 优化后的数据库结构不影响现有功能
- [x] 所有数据交互操作能正确执行
- [x] 测试用例全部通过（1142 passed）
- [x] 无编译错误和警告

## 修复建议

### 高优先级
1. 添加缺失的数据库表定义到迁移脚本
2. 修复 SQL 查询中的字段名不匹配问题
3. 统一时间戳字段命名

### 中优先级
1. 为布尔字段添加 is_* 前缀
2. 添加 serde alias 保持 API 兼容性

### 低优先级
1. 清理备份目录中的旧迁移文件
2. 更新 schema.sql 文件
