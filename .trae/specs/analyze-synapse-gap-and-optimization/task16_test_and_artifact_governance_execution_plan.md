# Task 16 - 测试体系与工作区产物治理执行文档

## 1. 目标

建立覆盖占位接口探测、schema 回归、路由契约测试的测试基线，并为 `tests/`、`docs/`、`artifacts/`、`test-results/` 和临时报表制定统一的工作区治理规则。

## 2. 范围

- 测试基线：占位接口探测、schema 回归、路由契约测试
- 目录治理：`tests/`、`docs/`、`artifacts/`、`test-results/`
- CI 产物：失败日志、测试报告、可下载产物、临时调试文件
- 非目标：本任务不直接重构所有历史测试文件

## 3. 输入

- `tasks.md` 中 `Task 16 详细说明`
- `Task 7` 验证与质量门禁方案
- `Task 11` 空壳接口治理方案
- `Task 15` schema gate 方案

## 4. 输出

- 三类测试基线定义
- 测试目录与命名规范
- 工作区产物分层规则
- CI 产物保存和清理策略

## 5. 执行阶段

### Phase 1: 测试基线设计

- 定义三类测试的目标、适用范围、最小样例和执行入口
- 约束断言必须关注真实业务数据，不只看状态码
- 标记可与 CI 直接挂钩的最小阻断集

### Phase 2: 目录与命名规范

- 盘点超大测试文件和散落测试
- 设计按能力域拆分的目录规则和命名规则
- 明确新增测试的归属原则

### Phase 3: 产物分层治理

- 将工作区内容划分为源码必审、文档必审、可生成报告、临时产物四类
- 明确每类内容的存放位置、生命周期和是否允许进入主干
- 规范临时报表命名与归档方式

### Phase 4: CI 产物策略

- 定义哪些产物常态上传，哪些仅失败时上传
- 定义保留时长、命名规则和排障要求
- 输出团队执行约定

## 6. 技术约束

- 测试基线必须强调真实数据断言
- 目录规范必须兼容当前 Rust workspace 结构
- 产物治理必须区分长期文档和一次性调试材料
- CI 策略必须说明诊断价值和保存期限

## 7. 里程碑与时限

- D1-D2：完成测试基线与现状问题盘点
- D3-D4：完成目录规范和产物治理设计
- D5：完成 CI 策略和团队执行约定

## 8. 质量指标

- 三类测试基线均有明确入口和样例
- 测试文件拆分规则清晰可执行
- 工作区产物分类覆盖率达到 100%
- 规则能显著减少评审噪音

## 9. 测试与验证

- 组织验证：抽样检查现有测试文件能否归类
- 产物验证：抽样检查典型文档和临时报表分类是否合理
- CI 验证：设计失败案例验证产物上传与命名策略
- 回归验证：确认与 Task 11/15 的规则不冲突

## 10. 风险与缓解

- 风险：目录规则过细，团队难以执行
- 缓解：优先覆盖高频场景，保留少量例外规则

- 风险：产物治理规则与现有习惯冲突
- 缓解：提供迁移期约定和示例模板

- 风险：CI 产物过多造成噪音
- 缓解：限制默认上传范围，仅在失败时扩展保留

## 11. 验收标准

- 已形成三类测试基线及其最小执行入口
- 已形成测试目录与命名规范
- 已形成工作区产物治理规则与 CI 策略
- 已形成面向团队执行的约定说明
