### 3.3 大表分区策略

#### events 表分区建议

events 表是最大的表，建议按时间范围分区：

```sql
-- 创建分区表
CREATE TABLE events_partitioned (
    event_id VARCHAR(255) NOT NULL,
    room_id VARCHAR(255) NOT NULL,
    sender VARCHAR(255) NOT NULL,
    event_type VARCHAR(255) NOT NULL,
    state_key VARCHAR(255),
    content JSONB NOT NULL DEFAULT '{}',
    origin_server_ts BIGINT NOT NULL,
    created_ts BIGINT NOT NULL,
    PRIMARY KEY (event_id, origin_server_ts)
) PARTITION BY RANGE (origin_server_ts);

-- 创建月度分区
CREATE TABLE events_2026_01 PARTITION OF events_partitioned
    FOR VALUES FROM (1704067200000) TO (1706745600000);

CREATE TABLE events_2026_02 PARTITION OF events_partitioned
    FOR VALUES FROM (1706745600000) TO (1709251200000);
```

## 七、最佳实践建议

### 7.1 索引设计原则

1. **选择性原则**：为高选择性列创建索引
2. **覆盖原则**：索引应覆盖常用查询
3. **复合索引顺序**：等值条件列在前，范围条件列在后
4. **避免过度索引**：索引会增加写入开销

### 7.2 查询优化原则

1. **避免 SELECT ***：明确列出所需字段
2. **使用 EXPLAIN ANALYZE**：分析查询计划
3. **合理使用 JOIN**：避免笛卡尔积
4. **分页查询**：使用 LIMIT + OFFSET 或游标

### 7.3 数据库维护

1. **定期 VACUUM**：清理死元组
2. **统计信息更新**：ANALYZE 命令
3. **索引重建**：REINDEX 命令
4. **监控告警**：设置性能监控
