# 字段变更管理流程

## 1. 变更申请

- 提交迁移前必须附带字段变更说明
- 说明必须包含字段语义、影响表、回滚方式、风险级别
- 必须更新 `field-mapping.csv` 中对应映射项

## 2. 评审清单

- 命名是否符合 `standards-spec.md`
- 数据类型是否与同语义字段一致
- 约束策略是否明确（外键、唯一、非空、检查）
- 是否存在冗余字段与兼容窗口设计

## 3. 发布门禁

- 通过字段审计脚本
- 通过迁移脚本 dry-run
- 通过关键业务 API 回归
- 通过数据完整性校验 SQL

## 4. 上线策略

- 按环境逐级发布：dev -> staging -> prod
- 迁移执行采用可重复、幂等 SQL
- 对高风险迁移采用分步切换

## 5. 回溯机制

- 每次发布保留前后两份审计报告
- 记录迁移版本、执行时间、责任人、回滚结果
- 异常回滚后必须补充根因分析与规则修订
