# 回滚方案与风险评估

## 已实施变更

- 00005：应用服务运行时兼容层（jsonb/text[] 与 appservice 字段兼容）
- 00006：应用服务字段标准化一期（回填、外键、部分约束）
- 00007：运行时兼容保护（撤销对旧写入路径不兼容的强约束）
- `db_migrate.sh`：兼容多版本 `schema_migrations` 结构
- 20260302000003 迁移：兼容不同 `schema_migrations` 列结构

## 风险分级

- 高风险：字段重命名、列类型转换、外键强校验
- 中风险：新增非空约束、默认值回填
- 低风险：新增索引、新增兼容函数、文档和审计流程

## 回滚策略

- 约束类回滚：`DROP CONSTRAINT`
- 非空回滚：`ALTER COLUMN DROP NOT NULL`
- 类型回滚：按映射表执行反向 `ALTER COLUMN TYPE`
- 迁移脚本回滚：按版本追加 guard 迁移，不做破坏性回滚
- 业务回滚：重启服务并切回旧查询路径

## 触发条件

- API 回归失败率显著升高
- 数据一致性检查出现新增不一致
- 核心写路径出现 5xx 或约束冲突

## 监控指标

- 迁移成功率
- 约束冲突次数
- 关键 API 错误率
- 核心查询 P95 延迟
