# synapse-rust 代码审查清单（占位实现与能力边界）

> 日期：2026-04-04  
> 目标：把“占位实现审计”的判定口径固化到审查流程，减少未来新增 placeholder/假成功的风险。

---

## 一、对外接口（路由层）

1. 如果端点对外宣称“已支持”，必须接入真实 service/storage 链路；禁止返回固定假数据或空对象误导能力判断。
2. 如果端点当前“不支持”，必须返回 Matrix 标准错误 `M_UNRECOGNIZED`，并补齐集成测试断言。
3. 对于 Matrix 规范允许的空成功响应（例如部分 `PUT/DELETE`），需要能在代码/测试中明确区分“规范行为”与“占位实现”。
4. 新增或修改路由时，至少补一个集成测试：
   - 路由存在性：未认证应返回 `401`（而不是 `404`）
   - 行为断言：成功/失败路径的 `errcode` 与关键字段

## 二、占位/未支持口径

1. 禁止引入 `placeholder` 相关字符串字面量到生产代码；如确有必要，仅允许出现在 `#[cfg(test)]` 测试模块内。
2. 新增“显式未支持接口”时，必须同步更新：
   - [UNSUPPORTED_ENDPOINTS.md](./UNSUPPORTED_ENDPOINTS.md)
   - 对应集成测试（断言 `M_UNRECOGNIZED`）
3. 对已有但未接通的预留实现（service/storage）：
   - 若会误导能力判断，必须改为显式未支持错误或移除对外可见的假返回
